О проекте

Медиформула - это e-commerce приложение, представляющее собой интернет-магазин, специализирующийся на продаже медицинской техники, изделий медицинского назначения, ортопедических товаров, товаров для здоровья и красоты, а также множества других товаров. Это приложение успешно интегрировано с API CommerceTools.

Наши инновации

Анонимный токен

Усилиями командной разработки внедрили создание анонимного токена, чтобы неавторизованный пользователь мог добавлять товары в корзину. При регистрации или авторизации происходит объединение данных из анонимной корзины с корзиной пользователя.

Развитие функциональности CommerceTools

Обнаружили, что в CommersTools API отсутствует возможность (или плохо искали) извлечь цену до применения промокода, которая нужна была для показа скидки пользователю. Благодаря командной разработке создали функцию, которая удаляет промокод, получает цену без промокода и записывает в стейт переменную, а затем снова добавляет промокод как было изначально, тем самым получали результат разницы между ценой с промокодом и без промокода.

Динамика координации команды

Для эффективной коммуникации и координации работы мы использовали платформy Telegram. Еженедельные созвоны позволили нам детально обсудить планы и стратегию, а 2 - 3 дополнительных созвона в течение недели помогали оперативно решать возникающие проблемы. Успешное общение проявилось в быстром решении сложных задач, обмене информацией о ходе работы и помощи в решении задач.

В процессе разработки мы применили гибкий подход, сочетая элементы Scrum и Kanban в рамках Agile-методологии.

Мы организовали работу в двухнедельных спринтах, как предполагает Scrum, что позволило нам концентрироваться на достижении конкретных целей и регулярно адаптироваться к изменениям.

Для визуализации рабочего процесса и управления потоком задач мы использовали GitLab Issue Boards, что улучшило прозрачность и эффективность работы. Такой комбинированный подход позволил нам сохранить структурированность и гибкость одновременно. Кроме того, каждый член команды имел четкое понимание текущих приоритетов и задач.

В качестве доски Канбан использовали встроенный инструмент GitLab Issue Boards

В нашем проекте мы используем современный технологический стек, основанный на React для разработки пользовательского интерфейса. Маршрутизацию реализуем с помощью библиотеки React Router Dom.

Для стилизации компонентов мы используем CSS, SASS и фреймворк Ant Design. Тестирование кода происходит с помощью Jest.

Для сборки проекта и разработки используется Webpack.

Кроме этого мы используем ESLint как инструмент статического анализа кода для выявления проблемных шаблонов, обнаруженных в коде JavaScript, Formik для работы с формами, Yup для валидации, Prettier для форматирования кода, которое нацелено на использование жёстко заданных правил по оформлению программ, Husky для упрощения работы с git hooks, Swiper для создания слайдеров на веб-страницах, Axios для HTTP-запросов и Yandex Maps для отображения карт.
